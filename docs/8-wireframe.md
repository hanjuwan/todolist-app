## 1. 개요 및 브레이크포인트 정책

### 1.1 와이어프레임 목적

본 문서는 개발 착수 전 화면 구조와 인터랙션 흐름을 시각화하여 개발자·기획자 간 공통 이해를 형성한다. 색상·폰트·정확한 여백 등 시각 디자인 세부 사항은 후속 UI 설계 단계에서 확정한다.

### 1.2 반응형 브레이크포인트

| 구분 | 범위 | 표기 |
|------|------|------|
| Mobile | < 768px | [Mobile] |
| Tablet | 768px ~ 1023px | (Desktop 레이아웃과 동일하게 처리, 일부 간격 조정) |
| Desktop | ≥ 1024px | [Desktop] |

각 화면마다 Desktop(≥1024px)과 Mobile(<768px) 두 가지 와이어프레임을 제공한다.

### 1.3 ASCII 표기 범례

| 표기 | 의미 |
|------|------|
| `[ 버튼명 ]` | 클릭 가능한 버튼 |
| `[___________]` | 텍스트 입력 필드 |
| `[v 옵션명   ]` | 드롭다운 셀렉트 |
| `[ ]` | 미선택 체크박스 |
| `[v]` | 선택된 체크박스 |
| `( )` | 미선택 라디오 버튼 |
| `(•)` | 선택된 라디오 버튼 |
| `[★]` / `[+]` | 아이콘 버튼 (FAB 등) |
| `...` | 콘텐츠 영역 (반복 요소 생략) |
| `─ ─ ─` | 점선 구분선 |

## 2. 공통 레이아웃 규약

### 2.1 Desktop 공통 네비게이션 (로그인 후)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  TodoListApp                      할일 목록  카테고리  마이페이지  [ 로그아웃 ]  │
└─────────────────────────────────────────────────────────────────────────┘
```

- 좌측: 로고/서비스명 (홈 링크)
- 우측: 주요 메뉴 링크 + 로그아웃 버튼
- 현재 화면의 메뉴 항목은 밑줄 또는 강조 표시

### 2.2 Mobile 공통 네비게이션 (로그인 후)

```
┌──────────────────────────┐
│  TodoListApp         [≡] │   ← 햄버거 메뉴 아이콘
└──────────────────────────┘

[≡] 탭 시 드로어(Drawer) 메뉴 표시:
┌──────────────────────────┐
│  TodoListApp         [✕] │
├──────────────────────────┤
│  > 할일 목록             │
│  > 카테고리 관리         │
│  > 마이페이지            │
├──────────────────────────┤
│  [ 로그아웃 ]            │
└──────────────────────────┘
```

### 2.3 비로그인 공통 네비게이션

```
┌─────────────────────────────────────────────────────────────────────────┐
│  TodoListApp                                      [ 로그인 ]  [ 회원가입 ] │
└─────────────────────────────────────────────────────────────────────────┘
```

| 컴포넌트 | 설명 |
|---------|------|
| 이메일 입력 | type="email", 자동완성 지원 |

| 로그인하기 버튼 | form submit, 로딩 중 비활성화 및 스피너 표시 |
| 오류 메시지 영역 | 로그인 실패 시 인라인 표시 ("이메일 또는 비밀번호가 올바르지 않습니다") |
| 회원가입하기 링크/버튼 | SCR-02로 이동 |

### 3.4 인터랙션 및 상태 변화

| 상태 | UI 변화 |
|------|---------|
| 기본 | 폼 초기 상태 |
| 로딩 중 | 로그인하기 버튼 비활성화, 스피너 표시 |
| 로그인 실패 (401) | 오류 메시지 표시, 비밀번호 필드 초기화 (SCN-12) |
| 네트워크 오류 | "서버 연결에 실패했습니다. 다시 시도해 주세요." 표시 |
| 로그인 성공 | JWT → Zustand 인메모리 저장 후 SCR-03으로 리다이렉트 |
| 세션 만료로 리다이렉트 | 상단 안내 배너: "세션이 만료되었습니다. 다시 로그인해 주세요." (SCN-13) |

```

│  [______________________________________] │
│                                          │

```

### 8.4 주요 컴포넌트

| 컴포넌트 | 설명 |
|---------|------|
| 이메일 표시 | 읽기 전용, 변경 불가 (변경 기능 미포함) |
| 이름 입력 | 수정 가능, 필수 |
| 현재 비밀번호 입력 | 비밀번호 변경 시 필수, show/hide 토글 |
| 새 비밀번호 입력 | 형식 유효성 힌트 표시, show/hide 토글 |
| 새 비밀번호 확인 입력 | 일치 여부 실시간 검사, show/hide 토글 |
| 변경사항 저장하기 버튼 | PATCH /api/users/me 호출 |
| 위험 구역 섹션 | 시각적으로 구분된 섹션 (경고 색상 또는 테두리 권장) |
| 회원 탈퇴 버튼 | 2단계 확인 다이얼로그 트리거 |

### 8.5 인터랙션 및 상태 변화

| 상태 | UI 변화 |
|------|---------|
| 초기 로드 | GET /api/users/me로 이름·이메일 표시, 비밀번호 필드 비움 |
| 이름만 수정 | 비밀번호 필드 비워도 이름만 변경 가능 |
| 비밀번호 변경 시 현재 비밀번호 오류 (401) | 현재 비밀번호 필드 하단 오류 메시지 (SCN-09) |
| 저장 성공 | "변경사항이 저장되었습니다." 토스트 메시지 |
| 탈퇴 다이얼로그 취소 | 다이얼로그 닫힘, 계정 유지 (SCN-10 Exception-1) |
| 탈퇴 성공 | Zustand 토큰 초기화 → SCR-01로 리다이렉트 + "탈퇴가 완료되었습니다." 안내 |
| 탈퇴 서버 오류 | "탈퇴 처리 중 오류가 발생했습니다." 표시, 트랜잭션 롤백으로 데이터 보전 (SCN-10 Exception-2) |

## 9. 디자인 토큰 메모

> 색상, 타이포그래피, 간격 등 구체적인 디자인 토큰 값은 후속 UI 설계 단계(Visual Design)에서 확정한다. 아래는 구조적 메모만 기록한다.

### 9.1 색상 시스템 (구조 예약, 값 미확정)

| 토큰 변수명 (예약) | 용도 | 확정 여부 |
|------------------|------|---------|
| `--color-primary` | 주요 버튼, 강조 요소 | 미확정 |
| `--color-primary-hover` | 주요 버튼 호버 상태 | 미확정 |
| `--color-danger` | 삭제 버튼, 위험 구역, 오류 메시지 | 미확정 |
| `--color-success` | 완료 체크박스, 성공 메시지 | 미확정 |
| `--color-warning` | 마감 임박 뱃지 등 경고 요소 | 미확정 |
| `--color-text-primary` | 주요 텍스트 | 미확정 |
| `--color-text-secondary` | 보조 텍스트, 힌트 | 미확정 |
| `--color-text-disabled` | 비활성 요소 | 미확정 |
| `--color-border` | 입력 필드, 카드 테두리 | 미확정 |
| `--color-bg` | 페이지 배경 | 미확정 |
| `--color-surface` | 카드, 모달 배경 | 미확정 |

> PRD 14장 및 프로젝트 구조 원칙 12장에 따라 1차부터 CSS 변수를 사용하여 2차 다크모드 확장 비용을 최소화한다.

### 9.2 타이포그래피 (구조 예약)

| 용도 | 크기 기준 | 확정 여부 |
|------|---------|---------|
| 페이지 제목 | 대형 | 미확정 |
| 섹션 제목 | 중형 | 미확정 |
| 본문 | 기본 | 미확정 |
| 보조 텍스트 / 뱃지 | 소형 | 미확정 |
| 오류 / 힌트 메시지 | 소형 | 미확정 |

### 9.3 간격 및 크기 (구조 예약)

| 항목 | 기준 | 확정 여부 |
|------|------|---------|
| 기본 간격 단위 | 4px 또는 8px 그리드 권장 | 미확정 |
| 터치 타깃 최소 크기 | 44×44px 이상 (SCN-14 성공 기준) | 방침 확정 |
| 카드 패딩 | 미확정 | 미확정 |
| 모달 최대 너비 (Desktop) | 미확정 | 미확정 |
| 네비게이션 높이 | 미확정 | 미확정 |

### 9.4 컴포넌트 상태 (구현 시 필요한 상태 목록)

모든 인터랙티브 컴포넌트(버튼, 입력, 체크박스, 드롭다운)는 아래 상태에 대한 시각적 처리가 필요하다.

- default (기본)
- hover (마우스 오버)
- focus (키보드 포커스 — 접근성 필수)
- active (클릭/탭 중)
- disabled (비활성)
- error (유효성 오류)
- loading (API 호출 중)
